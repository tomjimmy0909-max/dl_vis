## 〇、战略优先级（P0）

以下两项为**最高优先级**，其余条目在时间线上为此让路或与其对齐：

1. **画布端到端可运行**：Input + 隐藏层 + Output 与**数据管线**（数据源 → 批处理 → 对接 Input）在同一套图语义下可执行；当前为「画布 = 模型 + 对话框 = 数据」，见 [PRODUCT_DIRECTION.md §2–3](PRODUCT_DIRECTION.md)。
2. **图与代码双向连通**：DAG 级 **图 → PyTorch**；**PyTorch → 图**（优先 fx / 受限导入），与 [DESIGN.md §6](DESIGN.md) 对齐扩展。

## 一、项目概述

dl_vis 是一个基于 PyQt6 的深度学习可视化建模桌面工具，支持拖拽式构建神经网络计算图、导出 PyTorch 代码、训练与调试可视化。

### 核心定位
- **目标用户**：深度学习研究者、学生、需要快速原型设计的工程师
- **技术栈**：PyQt6 + PyTorch + Matplotlib
- **设计原则**：可视化优先、代码可导出、渐进式功能增强；**画布逐渐成为训练管线的单一编排面**

## 三、未来功能规划 📋

### 3.1 第二阶段（近期 - 1-2 个月）

#### 3.1.0 画布训练管线（P0，与 §〇 对齐）
| 功能 | 优先级 | 描述 |
|-----|-------|------|
| 数据 / DataLoader 节点 | 高 | 将 `.npy`/CSV/文件夹等挂载为图中节点，参数在画布上可视、可存 JSON |
| mini-batch 训练循环 | 高 | `runtime_torch` 由整图单批扩展为按 DataLoader 迭代 |
| Optimizer / LR 节点或统一配置 | 中 | 替代硬编码 Adam，语义可被导出到脚本 |
| 文档：画布 vs 对话框职责 | 中 | 在 USER/DESIGN 中写清迁移路径，避免用户「随便搭却跑不起来」 |
